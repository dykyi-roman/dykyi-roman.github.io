## How It Works

### Encoding

Each UUID (`xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`) contains 32 hex characters split into 5 segments. Every hex character (0-F) is encoded as a 2x2 dot matrix representing its 4 bits:

```
Bit layout per cell:        Examples:
[bit3] [bit2]               0 (0000) = ○○    F (1111) = ●●
[bit1] [bit0]                          ○○               ●●
```

The 32 cells are arranged in a 6x6 square grid (12x12 dots), with each UUID segment rendered in a distinct color:

| Segment | Characters | Default Color |
|---------|-----------|---------------|
| 1       | 8         | Red `#E74C3C` |
| 2       | 4         | Blue `#3498DB` |
| 3       | 4         | Green `#2ECC71` |
| 4       | 4         | Orange `#F39C12` |
| 5       | 12        | Purple `#9B59B6` |

### Grid Layout

```
Row 0: [seg1] [seg1] [seg1] [seg1] [seg1] [seg1]
Row 1: [seg1] [seg1] [seg2] [seg2] [seg2] [seg2]
Row 2: [seg3] [seg3] [seg3] [seg3] [seg4] [seg4]
Row 3: [seg4] [seg4] [seg5] [seg5] [seg5] [seg5]
Row 4: [seg5] [seg5] [seg5] [seg5] [seg5] [seg5]
Row 5: [seg5] [seg5] [    ] [    ] [    ] [    ]
```

Each cell is a 2x2 dot matrix. The final 4 positions are padding (empty dots). The output is a compact **216x216 px** square image.

### Decoding

Upload a previously generated PNG image, and the decoder will:
1. Verify the image matches the expected square dimensions
2. Sample pixel colors at each dot center
3. Classify each dot as filled or empty based on brightness
4. Reconstruct the UUID from the 2x2 bit matrices

## Features

- **Generate** — create random UUID v4 (using `crypto.getRandomValues()`)
- **Encode** — convert any valid UUID into a colored dot-matrix picture
- **Customize** — pick individual colors for each of the 5 UUID segments
- **Export** — download as PNG or SVG
- **Decode** — drag & drop or upload a PNG to extract the UUID back
- **Copy** — one-click clipboard copy of decoded UUIDs
- **Responsive** — works on desktop and mobile

## Getting Started

No build step required. Just open the file in a browser:

```bash
# Option 1: directly
open index.html

# Option 2: with a local server
npx serve .
# or
python3 -m http.server 8000
```

## Project Structure

```
uuid-to-picture/
├── index.html              — Main HTML page
├── css/
│   └── style.css           — Styles (responsive, minimal)
└── js/
    ├── uuid.js             — UUID validation, parsing, generation
    ├── encoder.js          — UUID → GridModel (logical dot matrix)
    ├── canvas-renderer.js  — GridModel → Canvas (display + PNG export)
    ├── svg-renderer.js     — GridModel → SVG string (SVG export)
    ├── decoder.js          — PNG image → UUID (pixel sampling)
    └── app.js              — Entry point, UI event handlers
```

## Roundtrip Verification

The encoding is fully reversible for images generated by this application:

1. Enter or generate a UUID
2. Click **Generate Picture**
3. Click **Download PNG**
4. Drag the downloaded PNG into the **Picture → UUID** section
5. The decoded UUID matches the original

## Browser Support

Works in all modern browsers that support:
- `<canvas>` 2D context
- `crypto.getRandomValues()`
- `navigator.clipboard` API (with fallback)
- Drag & Drop API
